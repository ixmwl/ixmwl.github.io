CSS基础--选择器

发布 : 2019-04-14 分类 : 前端 浏览 :

选择器是什么？

每一条css样式定义由两部分组成，形式如下： 

选择器{样式}

在{}之前的部分就是“选择器”。 “选择器”指明了{}中的“样式”的作用对象，也就是“样式”作用于网页中的哪些元素

基础选择器

标签选择器

之前html介绍中有总结一个常用的标签表格，此处再写一遍，加深下记忆

标签名称 标签作用
html 包裹所有的其他标签，是html最外层的元素
head 文档的头部
title 表示网页的标题
body 所有的网页内容都放在body中
div 表示一个容器或者说一个盒子
span 功能和div类似，span可以用来装一小段文字 和div的区别就是 在一行内显示
h1-h6 表示标题 数字越大 字体越小
p 表示段落标签
hr 表示一条水平分割线
ul li 表示无序列表
i/em 表示斜体
strong 表示加粗
br 表示换行
img 表示图像标签，可以引入图像 属性：特征，在开始标签后面 以属性名称=“属性值” 这种形式表示给html标签加属性,img标签必须加上src属性才能把图像展示出来。
<img src="imageName.png">
alt属性： 当图片没有办法显示的时候用来提示用户的文字
title属性：当鼠标移动到图片的时候，用来提示用户的文字
a 表示超链接，可以链接一切资源（包括网址 视频 文字 等等）
href属性： 超链接的地址
target属性： _blank 表示新建一个浏览器标签页来显示超链接的内容，不会覆盖原来的网页

此处的标签选择器要用到的就是上面表格中展示的标签了，所以根据css样式定义会有如下例子：

<head>
    <title>css</title>
    <style type="text/css">
        p {
            font-size: 14px;
        }
    </style>
</head>
<body>
   <p>css</p>
</body>

在head的样式中p就是标签，这种设置会针对全局的p标签，如果其他标签未设置样式则都会沿用这个标签选择器的样式

「总结」

  1. 所有的标签，都可以是选择器。比如ul、li、label、dt、dl、input、div等。
  2. 无论这个标签藏的多深，一定能够被选择上。
  3. 选择的所有，而不是一个。

ID选择器

ID选择器记住用#来定义，而且是针对某一个特定的标签来使用，只能使用一次，不能重复。例如：

<head>
    <title>Document</title>
    <style type="text/css">
        #csstitle {
            border: 3px dashed green;
        }
    </style>
</head>

然后在body中如果需要用到此样式，直接将这个id赋给标签即可

<body>
    <p id="csstitle">css</p>
</body>

「总结」
任何的HTML标签都可以有id属性。表示这个标签的名字。这个标签的名字，可以任取，但是：

  1. 只能有字母、数字、下划线。
  2. 必须以字母开头。
  3. 不能和标签同名。比如id不能叫做body、img、a。
  4. 大小写严格区分，也就是说aa,和AA是两个不同的ID

另外，特别强调的是：HTML页面，不能出现相同的id，哪怕他们不是一个类型。比如页面上有一个id为title的p，一个id为title的div，是非法的！

一个标签可以被多个选择器选择，比如上面已经讲到了标签选择器和ID选择器。那么我们可以同时让标签选择器和id选择器作用于同一个标签。例如：

<head>
    <title>Document</title>
    <style type="text/css">
        p {
            font-size: 40px;
        }

        #csstitle {
            border: 3px dashed green;
        }
    </style>
</head>>

<body>
    <p id="csstitle">css</p>
</body>

如上所述，标签选择器和ID选择器都作用于p标签上

类选择器

css中用.来定义类选择器,和id非常相似，任何的标签都可以携带id属性和class属性,可以灵活的作用于你所想要的所有标签上。

<head>
    <title>Document</title>
    <style type="text/css">
        .sub-title{
            height: 100px;
            background-color: bisque;
        }
    </style>
</head>

根据每个标签都有class属性，那么我们可以在别处使用class来引用这个样式

<body>
    <p class="sub-title">css</p>
</body>
  1. 类选择器可以被多种标签使用
  2. 同一个标签可以使用多个类选择器，用空格分开即可，如 <h class="classone classtwo">我是一个标题啊</h>

「总结」
类选择器的使用能够决定一个人对于css使用的水平，因此关于class选择器有以下需要注意：

  1. 尽可能标签要多携带几个类，共同完成这个标签的样式。不要试图一个类完成这个标签的所有样式
  2. 每个类尽可能的小，以便其他地方可以使用，实现公共的作用
  3. 具体css编码中尽量使用class选择器，因为id感觉在js中用的更多

通配选择器

通配选择器使用*定义，单独使用时，这个选择器可以与文档中的任何元素匹配，就像一个通配符。
虽然通配选择器的功能强大，但是出于效率考虑，很少有人使用它。所以暂时就这样，后续再详细去了解。

高级选择器

相邻选择器

相邻选择器可以选择紧跟其后的相邻元素，注意：相邻选择器用+来定义连接

<head>
    <title>相邻选择器</title>
    <style type="text/css">
        h1+div{
            width: 100px;
            height: 100px;
            background-color: blue;
        }
    </style>
</head>

<body>
    <h1>相邻选择器测试</h1>
    <div id="box1"></div>
</body>

多元素选择器

当几个元素有共同属性的时候，可以使用多元素选择器，也就是同时为多个元素设置样式，用,逗号隔开

<head>
    <title>多元素选择器</title>
    <style type="text/css">
        h1,h2,h3,h4,p{
            background-color: red;
        }
    </style>
</head>

后代选择器

后代选择器作用于父元素下的所有子元素，用空格隔开

<head>
    <title>后代选择器</title>
    <style type="text/css">
        #box p{
            background-color: red;
        }
    </style>
</head>

<body>
    <div id="box">
        <p>后代选择器</p>
        <p>这是一个p标签</p>
        <div id="box1">
            <p>这是另一个div下的p 标签</p>
        </div>
    </div>
</body>

如上可以把id为box的div下的所有p标签背景设置为红色，包括id为box1的div下的p标签。

「总结」

两个标签不一定是连续紧挨着的，只要保持一个后代的关联即可。也就是说，选择的是后代，不一定是子代，也可能是孙代。

子元素选择器

子元素选择器作用于父元素的子元素，子元素选择器与后代选择器的区别在于：后者可以作用于子孙元素，但是前者只能作用于子元素，不能跨代。用>来定义，例子还是用上面后代选择器的例子稍作修改，可以类比看下区别：

<head>
    <title>后代选择器</title>
    <style type="text/css">
        #box>p{
            background-color: red;
        }
    </style>
</head>

<body>
    <div id="box">
        <p>后代选择器</p>
        <p>这是一个p标签</p>
        <div id="box1">
            <p>这是另一个div下的p 标签</p>
        </div>
    </div>
</body>

此作用结果就是id为box的子代p标签背景都变成红色，但是id为box1的div中的p孙代标签背景是没有被设置的。

属性选择器

E[attr]匹配所有具有attr属性的E元素，不考虑他的值，例如：input[name],只要有name属性的input元素都会被选中。

<head>
    <title>后代选择器</title>
    <style type="text/css">
        input[name]{
            border: red solid 1px;
        }
    </style>
</head>

<body>
    <form>
        <input type="text" name="input1">
        <input type="text" name="input2">
        <input type="submit" value="提交">
    </form>
</body>

运行结果，拥有name属性的两个input元素边框被设置成了红色，最后一个并没有。

伪类和伪元素

伪类是添加到选择器的关键字，指定要选择的元素的特殊状态。例如，:hover 可被用于在用户将鼠标悬停在按钮上时改变按钮的颜色。语法如下：

selector:pseudo-class {
  property: value;
}

a标签的4种状态

link:未访问链接
hover:鼠标放上状态，鼠标放到某个标签上的时候
visited:已访问链接，
active:当前活动链接，点击某个标签没有松鼠标时

CSS允许对于元素的不同状态，定义不同的样式信息。伪类选择器又分为两种：

静态伪类：只能用于超链接
动态伪类：针对所有标签都适用

before&after

这个两个伪元素在真正页面元素内部之前和之后添加新内容

<head>
    <meta charset="UTF-8">
    <title>伪类</title>
    <style type="text/css">
        a:before{
           content: '点我';
        } 
        a:after{
            content: '警告';
        }
    </style>
</head>
<body>
    <form>
        <a href="https://www.baidu.com">百度</a>
        <p>百度以下，就知道了！</p>
        <a href="https://www.pgyer.com">蒲公英分发平台</a>
    </form>
</body>

例如上面的代码就是在a标签前后添加了点我和警告

「总结」

  1. 伪元素不属于文档，所以js无法操作它
  2. 伪元素属于主元素的一部分，因此点击伪元素触发的是主元素的click事件
  3. 优点：减少dom节点、让css帮助解决部分js问题，让问题变得简单
  4. 缺点：不利于SEO，无法审查元素，不利于调试
