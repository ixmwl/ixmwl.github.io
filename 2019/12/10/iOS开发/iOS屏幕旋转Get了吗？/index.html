iOS屏幕旋转Get了吗

在实际开发过程中，总会遇到各种各样的需求，关于屏幕旋转的适配，也会经常遇到。比如以下要求：

  1. 所有页面都不能旋转
  2. 某一个页面可以旋转
  3. 或者只支持左或者右旋转

除了旋转需求外，旋转过后的屏幕适配等也是一个让人头疼的问题，总有一款会让你抓狂。此文暂时就只介绍下屏幕旋转的相关操作，面对各种旋转需求，我们改如何选择。

配置旋转属性

关于屏幕旋转属性的配置有三种选择：

  • Targets中设置
  • info.plist配置项
  • AppDelegate中代码设置

在说旋转前，有两个枚举类需要介绍下UIDeviceOrientation和UIDeviceOrientation

UIDeviceOrientation 设备方向
UIInterfaceOrientation 页面方向

设备方向只能取值，不能设置，经常用于检测设备方向变化。再对照看下页面方向其实很多都是能对应上的，但有一点需要注意的是：

UIInterfaceOrientationLandscapeLeft 与UIDeviceOrientationLandscapeRight相等，反之亦然

而且设备方向多了个屏幕朝上还是朝下，这对于程序页面来说是没啥意义的。说完这两个的区别与联系，下面我们就一一来看下，具体该如何配置。

Targets中配置

在调试屏幕旋转前，去工程中下面的路径中的旋转方向选项勾选上，否则你后面的任何代码可能都对屏幕旋转没有任何影响Targets>General>Deployment Info>Device Orientation

  • Portrait
  • Upside Down
  • Landscape Left
  • Landscape Right

info.plist配置

当我们设置了Device Orientation之后，再到info.plist中查看Supported interface orientation,我们会看到info.plist已经自动生成了配置项如下

其实info.plist和targets中的配置是同步的，所以两种方式可以理解为一个方法。

AppDelegate代码

因为appdelegate所持有的window对象是全局的，所以在此处去设置屏幕旋转也是可以全局有效的

- (UIInterfaceOrientationMask)application:(UIApplication *)application supportedInterfaceOrientationsForWindow:(UIWindow *)window {
    return  UIInterfaceOrientationMaskAll;
}

return的部分可供选择的在前面UIInterfaceOrientation和UIDeviceOrientation的比较中已经枚举过。另外值得注意的是，如果你在delegate中设置了屏幕旋转，那么前面两个的设置将会无效，一切以delegate中代码的设置为准。

旋转的优先级

综上可见旋转设置有很多，那么具体如何设置才会生效，其中就存在一个优先级的问题，一是配置的优先级：

Appdelegate&&Window > 工程Target属性配置

二是应用视图层级的优先级，项目中常见的就是以UITabbarController作为Window的根视图控制器，然后管理着若干个导航控制器UINavigationController，再由导航栏控制器去管理普通的视图控制器UIViewController，那么以此为例，优先级如下：

UITabbarController > UINavigationController > UIViewController

如果具有高优先级的控制器关闭了旋转的功能，那么低优先级的控制器也是无法旋转的，除非强制处理。另外综上

配置的优先级 > 应用视图层级的优先级

如果全局配置都没开启，那么视图层级上面也是不会旋转的

实现旋转功能

在iOS的控制中心大家可以找到一个可以主动锁定横竖屏的按钮，如下所示：

当此功能开启时，那么不做特殊处理，程序中所有的允许横竖屏切换的设置都会无效，因为此功能开启理论上系统是禁止横竖屏切换的。当然也有例外，前面也说了特殊处理、理论上，所以就算此功能开启，我们也有相关黑科技去强制横竖屏，后面再细细道来。

其实大多数情况下，开发app过程中大多数都是竖屏，少数情况下有部分页面有需求要横屏。横屏可能在游戏类应用中见到的更多。

关于设置屏幕旋转，主要是系统提供的一下几个方法：

-(BOOL)shouldAutorotate{
    return YES或者NO;
}

//支持哪些屏幕方向
- (UIInterfaceOrientationMask)supportedInterfaceOrientations {
    return 支持方向;
}

//默认方向
- (UIInterfaceOrientation)preferredInterfaceOrientationForPresentation{
    return 默认方向;
}

shouldAutorotate 返回YES，则表明此页面是支持旋转的，但最终是否会旋转取决于其上级，身不由己。

就上面的讲优先级的时候，说到的层级，具体实现可以如下：

UITabbarController

//是否自动旋转
-(BOOL)shouldAutorotate{
    return self.selectedViewController.shouldAutorotate;
}

//支持哪些屏幕方向
- (UIInterfaceOrientationMask)supportedInterfaceOrientations {
    return [self.selectedViewController supportedInterfaceOrientations];
}

//默认方向
- (UIInterfaceOrientation)preferredInterfaceOrientationForPresentation{
    return [self.selectedViewController preferredInterfaceOrientationForPresentation];
}

UINavigationController

//返回导航控制器的顶层视图控制器的自动旋转属性，因为导航控制器是以栈的原因叠加VC的
//topViewController是其最顶层的视图控制器，
-(BOOL)shouldAutorotate{
    return self.topViewController.shouldAutorotate;
}

//支持哪些屏幕方向
- (UIInterfaceOrientationMask)supportedInterfaceOrientations {
    return [self.topViewController supportedInterfaceOrientations];
}

//默认方向
- (UIInterfaceOrientation)preferredInterfaceOrientationForPresentation{
    return [self.topViewController preferredInterfaceOrientationForPresentation];
}

强制旋转

假如控制中心的锁定开启，那么此时如上代码中的设置已经无法起到控制作用了，而且以下的旋转监听也无法拦截到任何信息,因为此时屏幕无法旋转。

[[NSNotificationCenter defaultCenter]addObserver:self selector:@selector(didChangeRotate:) name:UIApplicationDidChangeStatusBarFrameNotification object:nil];


- (void)didChangeRotate:(NSNotification*)notice {
    if ([[UIDevice currentDevice] orientation] == UIInterfaceOrientationPortrait){
        //竖屏
    } else {   
    }
}

那么如果需要部分页面仍然能旋转切换，此时我们就要借助CoreMotion，根据陀螺仪计算判断当前屏幕的横竖屏状态，然后去强制切换。具体强制切换横竖屏的代码如下：

- (void)interfaceOrientation:(UIInterfaceOrientation)orientation {
    if ([[UIDevice currentDevice] respondsToSelector:@selector(setOrientation:)]) {
        SEL selector             = NSSelectorFromString(@"setOrientation:");
        NSInvocation *invocation = [NSInvocation invocationWithMethodSignature:[UIDevice instanceMethodSignatureForSelector:selector]];
        [invocation setSelector:selector];
        [invocation setTarget:[UIDevice currentDevice]];
        int val                  = orientation;
        [invocation setArgument:&val atIndex:2];
        [invocation invoke];
    }
}

后续我们就需要根据CoreMotion去判断状态，将屏幕状态传给上述方法，即可在锁定状态下的很竖屏主动切换。此处监控的代码具体可前往github查看，【传送门】
